Algorithm Comparison

RSA-PSS

Probabilistic Signature Scheme - recommended for new applications

Sig size: 256-512 bytes | Security: Based on RSA hardness

DSA

Digital Signature Algorithm - US Federal Standard

Sig size: 320 bytes | Key: 1024-3072 bits

ECDSA

Elliptic Curve DSA - smaller keys, equivalent security

Sig size: 64-148 bytes | Key: 224-521 bits

Security Properties

✓ Authenticity: Verifies sender identity
✓ Integrity: Ensures message not modified
✓ Non-repudiation: Sender cannot deny signing
! Confidentiality: Not provided (use encryption)

Attack Scenarios

Reused k (ECDSA/DSA): If random k is reused, private key can be computed

Bleichenbacher: RSA-PKCS#1 vulnerable to padding oracle attacks

Countermeasure: Use PSS padding, unique k per signature
