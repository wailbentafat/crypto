Algorithm Reference

RSA-PSS

Probabilistic Signature Scheme. Uses random salt — same message produces different signatures each time.

Key: 2048–4096 bits
Sig: 256–512 bytes
Recommended for new systems

RSA-PKCS#1 v1.5

Deterministic — same message always produces same signature. Vulnerable to Bleichenbacher attack if misused.

Key: 2048–4096 bits
Sig: 256–512 bytes
Legacy — prefer PSS

ECDSA P-256

Elliptic Curve DSA on NIST P-256. Much shorter keys than RSA for equivalent security.

Key: 256 bits
Sig: ~72 bytes
Used in TLS, Bitcoin, SSH

ECDSA P-384

Higher security variant. Preferred for government and long-term security requirements (NSA Suite B).

Key: 384 bits
Sig: ~104 bytes
NSA Suite B compliant

Security Properties

✓Authenticity — proves the signer's identity (only they have the private key)
✓Integrity — any modification to the message breaks the signature
✓Non-repudiation — signer cannot later deny having signed
✗Confidentiality — signatures do not encrypt; use encryption separately

Known Attack Scenarios

Reused k (ECDSA/DSA)

If the random nonce k is reused across two signatures, the private key can be mathematically extracted. Sony PS3 was broken this way.

Bleichenbacher (PKCS#1 v1.5)

Adaptive chosen-ciphertext attack on RSA PKCS#1 padding. Mitigated by using PSS padding instead.
